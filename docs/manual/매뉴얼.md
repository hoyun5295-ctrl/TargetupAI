## 🚦 다음 세션 재개 Step-by-step

### 서팀장 캡처 자산이 도착했을 때

```
1. 받은 스크린샷 파일을 확인 → 챕터별로 폴더 분류
   docs/manual/captures/vol-01/...
   docs/manual/captures/vol-02/...
   ...

2. 각 캡처의 파일명을 촬영 대본 cap-ID에 매핑 (필요시 shots/vol-XX-script.md 업데이트)

3. 첫 시범 제작: Vol.01
   a. docs/manual/chapters/vol-01-content.md 작성
      — § 01~06 본문, 캡션, Callout, Meta row 내용
   b. docs/manual/chapters/vol-01-prompt.txt 작성
      — Claude Design에 그대로 복붙할 프롬프트
      — EDITORIAL-SYSTEM.md §8 스켈레톤 기반
      — 참조 이미지 경로 명시 (transition.html 렌더본 + 캡처 경로)
   c. Harold님이 Claude Design에서 실행 → HTML 수령 →
      docs/manual/chapters/vol-01.html 로 저장
   d. 검수 후 피드백 → 프롬프트 다듬어 재생성

4. Vol.01 톤 확정되면 Vol.02~08 동일 패턴 반복
   — EDITORIAL-SYSTEM.md가 이미 확정되어 있어 프롬프트는 80% 재활용

5. 마지막에 docs/manual/chapters/index.html (매뉴얼 허브) 제작
```

### Harold님이 "계속 진행해" 같은 간단한 명령만 내리셨을 때

1. 이 문서 먼저 읽기
2. `docs/manual/captures/` 폴더에 스샷 도착 여부 확인
3. 도착했으면 위 Step 1부터
4. 아직이면 대기 유지 + 현재 상태 보고

## 🔄 워크플로우 (Claude Design 기반)

Harold님이 transition.html 만드실 때 썼던 **Anthropic Labs의 Claude Design (Research Preview)** 툴을 이번 매뉴얼에도 활용.

| 역할 | 담당 | 산출물 |
|------|------|--------|
| 콘텐츠 기획 | 제(Claude Code) | `vol-XX-content.md` (텍스트·톤·섹션 구조) |
| 프롬프트 작성 | 제(Claude Code) | `vol-XX-prompt.txt` (Claude Design 입력용) |
| 참조 이미지 준비 | 제 + Harold님 | (a) transition.html 렌더 스크린샷 1~2장 (b) 서팀장 촬영 앱 캡처 |
| HTML 생성 | **Claude Design** | `vol-XX.html` (최종 매뉴얼) |
| 복붙 실행 | Harold님 | Claude Design 앱에서 프롬프트 + 이미지 붙여넣기 |
| 검수/피드백 | Harold님 + 제 | 프롬프트 미세 조정 후 재생성 |

**참고:** transition.html은 Harold님이 Claude Design에 "popup" 프로젝트로 프롬프트 + 팝업 이미지 1장만 주고 뽑아낸 결과. 즉 **프롬프트 품질이 결과의 전부**.
