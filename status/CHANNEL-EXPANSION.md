## 1. 현재 발송 채널 구조

| 채널 | 위치 | 상태 |
|------|------|------|
| SMS / LMS / MMS | 메인 발송창 (문자 탭) | ✅ 운영 중 |
| 카카오 브랜드MSG | 메인 발송창 (브랜드MSG 탭) | ✅ 운영 중 |
| 카카오 알림톡 | 메인 발송창 (알림톡 탭) | ✅ 운영 중 |

## 2. 변경 계획 (Harold님 구상)

### 2-1. 메인 발송창 채널 변경
| AS-IS | TO-BE |
|-------|-------|
| 문자 / 브랜드MSG / 알림톡 | 문자 / **알림톡** / **RCS** |

- **브랜드MSG** → 별도 메뉴(카카오)의 전용 모달로 이동
- **알림톡** → 메인 발송창에 유지
- **RCS** → 메인 발송창에 새로 추가

### 2-2. 메인메뉴 구성
```
AI분석 / 자동발송 / [카카오] / 직접발송
```
- **카카오 메뉴 (신규)**: 직접발송 왼쪽에 추가
  - 브랜드메시지 발송
  - 알림톡 템플릿 관리

## 3. RCS 연동 (젬텍 NGS)

### 3-1. 연동 규격
- **규격서**: `젬텍_NGS_RCS_연동규격서_V2.0.2.pdf` (Harold님 다운로드 폴더)
- **연동 방식**: QTmsg(젬텍 NGS) API 연동
- **지원 메시지 유형**: RCS SMS, RCS LMS, RCS MMS, RCS 템플릿

### 3-2. RCS 특징
- SMS/LMS의 리치 미디어 대체 채널
- 수신 단말이 RCS 미지원 시 자동 SMS/LMS 폴백
- 발신번호 기반 (기존 SMS 인프라와 호환)
- 브랜드 프로필, 버튼, 카드 등 리치 UI 지원

### 3-3. 구현 시 고려사항
- MySQL QTmsg 큐에 RCS 메시지 타입 INSERT (기존 SMS 큐 재활용 가능 여부 확인)
- 발송 결과 동기화 (sync-results에 RCS 결과 매핑 추가)
- CT-04(sms-queue.ts)에 RCS 큐 조작 함수 추가
- 프론트엔드 발송창에 RCS 탭 + UI

## 4. 카카오 알림톡 API

### 4-1. 연동 규격
- **API 연동사**: 휴머스온 (HumansOn)
- **인증**: 휴머스온 계정 기반
- *(참고: Bizppurio API manual v3.9 — 알림톡 템플릿 규격 참조용)*

### 4-2. 알림톡 특징
- 템플릿 사전 등록 + 카카오 검수 필요
- 수신자가 카카오톡 미사용 시 SMS 폴백 가능
- 버튼(웹링크, 앱링크, 배송조회 등) 지원
- 이미지 + 하이라이트 템플릿 지원

### 4-3. 알림톡 템플릿 관리 (카카오 메뉴에 포함)
- 템플릿 CRUD (생성/수정/삭제)
- 카카오 검수 상태 관리 (대기/승인/반려)
- 템플릿 카테고리별 분류
- 변수 치환 미리보기

## 5. 브랜드메시지 별도 모달 (카카오 메뉴에 포함)

### 5-1. 분리 사유
- 브랜드메시지 UI가 SMS/LMS와 완전히 다름 (버튼, 카드, 이미지 등)
- 발송 파이프라인도 별도 (카카오 API)
- 메인 발송창에 넣기에는 복잡도가 너무 높음

### 5-2. 구성
- 브랜드메시지 전용 발송 모달
- 발신 프로필 선택
- 버블 타입 선택 (TEXT, IMAGE, WIDE 등)
- 타겟팅 (I/M/N)
- 기존 카카오 발송 파이프라인 재활용
