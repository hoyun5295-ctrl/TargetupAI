# 전단AI — 한줄로 AI 전단지 솔루션 설계 문서

> **최초 작성:** 2026-03-28 (Harold님 아이디어)
> **상태:** 프로토타입 완성 — 다음 세션에서 백엔드 API + 별도 프론트 구현
> **최종 업데이트:** 2026-03-28

---

## 1. 배경 및 기회

### 현재 시장 상황
- 마트/로컬 매장들이 SMS/LMS로 행사 상품을 텍스트 나열하여 발송
- 예시: `(광고)데레사 짭짤이-9900 청송사과20K-79800 양배추-990 js.ddr.kr 무료수신거부 0808804934`
- 단축 URL(`js.ddr.kr`)을 넣어 클릭하면 전단지 이미지가 로딩되는 구조
- 현재는 전단지 이미지를 별도로 제작해서 외부 호스팅 → 번거로움

### 한줄로 AI 기회
- 상품/가격 입력만 하면 **예쁜 전단지 페이지 자동생성 + 단축 URL + SMS 발송**을 원스톱으로 제공
- 마트뿐 아니라 **빵집, 정육점, 꽃집, 카페, 미용실** 등 로컬 매장 전체로 확장 가능
- 한줄로 플랫폼 안의 **미니 플랫폼** (전단AI)

## 2. 핵심 컨셉

### "전단AI" — 한줄로 AI의 가벼운 전단지 솔루션

**아키텍처 결정 (2026-03-28 확정):**
- **백엔드:** 한줄로 기존 코드베이스 재활용 (DB, 발송, 수신거부, 과금)
- **프론트엔드:** 별도 앱으로 분리 (마트/로컬 매장 전용 심플 UI)
- **이유:** 기존 한줄로는 법인/기업용이라 복잡한 기능이 많아 함께 두면 지저분함. 마트 사장님 전용 간단한 화면이 필요

**업종 범위 (2026-03-28 확정):** 마트부터 시작 → 점진적으로 확장

**업종별 특화 가능:**
- 마트/슈퍼: 카테고리별 상품 + 할인가 (청과, 공산, 축산, 수산)
- 카페/베이커리: 신메뉴, 시즌 한정
- 미용실: 이달의 할인, 시술 메뉴
- 정육점/수산: 오늘의 특가
- 꽃집: 시즌 꽃다발, 이벤트

## 3. 사용자 플로우 (마트 기준)

```
1. 한줄로 대시보드 → "전단AI" 메뉴 클릭
2. 업종 선택 (마트/카페/미용실 등) — 업종별 템플릿 제공
3. 행사 정보 입력:
   - 행사명: "26년 3월 데레사 행사"
   - 기간: 3/28 ~ 4/3
   - 카테고리 추가: [청과/야채] [공산] [축산] [수산] ...
   - 상품 입력: 상품명 / 원가 / 할인가 (테이블 형태, 행 추가/삭제)
   - 매장 로고/이미지 (선택)
4. AI 자동 처리:
   - 전단지 웹페이지 자동생성 (모바일 최적화)
   - 단축 URL 발급 (예: hanjul-flyer.kr/abc123)
   - SMS 문구 자동 추천: "(광고)데레사 3월 행사 딸기6980 사과79800 ▶ hanjul-flyer.kr/abc123"
5. 수신자 선택 (기존 한줄로 기능 재활용):
   - 고객DB 전체 / 필터 타겟 / 파일 업로드
6. 발송 (기존 발송 파이프라인 재활용)
7. 결과 확인:
   - 발송 성공/실패 (기존)
   - ★ 전단지 클릭률 (단축 URL 추적) — 신규
   - ★ 어떤 상품이 가장 관심 높은지 (상품별 클릭 영역 추적 — 향후)
```

## 4. 기술 설계 (초안)

### 4-1. 단축 URL 시스템

| 구성 | 설명 |
|------|------|
| **DB 테이블** | `short_urls` — code(unique), original_url, company_id, campaign_id, flyer_id, created_at, expires_at |
| **리다이렉트 API** | `GET /s/:code` → 302 redirect + 클릭 로그 저장 |
| **클릭 로그 테이블** | `url_clicks` — short_url_id, clicked_at, ip, user_agent, referer |
| **코드 생성** | nanoid 6~8자 (예: `hanjul-flyer.kr/k3x9mT`) |
| **만료** | 기본 90일 (마트 행사는 보통 1~2주이므로 충분) |

### 4-2. 전단지 페이지 시스템

| 구성 | 설명 |
|------|------|
| **DB 테이블** | `flyers` — company_id, title, period_start, period_end, categories(JSONB), items(JSONB), template, logo_url, status |
| **렌더링** | 서버사이드 또는 SPA — 모바일 최적화 HTML 페이지 |
| **접근** | `GET /f/:code` → 전단지 페이지 렌더링 (단축 URL이 여기로 리다이렉트) |
| **템플릿** | 업종별 디자인 템플릿 (마트, 카페, 미용실 등) |
| **이미지** | 상품 이미지 업로드 선택사항 (없어도 텍스트 전단지로 생성) |

### 4-3. 전단지 데이터 구조 (JSONB 예시)

```json

  "logo_url": "/uploads/logo_deresa.png",
```

### 4-4. 기존 한줄로 파이프라인 재활용

| 기존 기능 | 전단AI에서 재활용 |
|-----------|-------------------|
| 고객DB + 필터 | 수신자 선택 |
| SMS/LMS 발송 파이프라인 | 전단지 URL 포함 메시지 발송 |
| 수신거부 (080) | 그대로 적용 |
| 선불 잔액 차감 | 그대로 적용 |
| 발송 결과/통계 | + 클릭률 추가 |
| 브랜드 격리 (store_code) | 매장별 전단지 분리 |

## 5. AI 활용 포인트

1. **SMS 문구 자동생성:** 입력된 상품 중 핵심 3~4개 자동 선별 → 90byte 이내 SMS 문구 추천
2. **전단지 레이아웃 추천:** 상품 수/카테고리에 따라 최적 레이아웃 자동 선택
3. **발송 시간 추천:** "마트 고객은 목/금 오전에 클릭률이 높습니다" (향후 데이터 축적 후)
4. **상품명 자동보정:** "짭짤이토마토1박스" → "짭짤이 토마토 1박스" 띄어쓰기/포맷 정리

---

## 6. 수익 모델

- 기존 SMS/LMS 발송 과금 그대로 (단가 동일)
- 전단지 생성 자체는 **무료 or 프로 이상** (요금제 게이팅)
- 단축 URL + 클릭 추적은 부가 가치 (프로 이상)
- 마트 전용 패키지로 묶어서 영업 가능

## 10. 미결정 사항

1. **고객 DB 관리 방식:** 전단AI 자체 고객DB 메뉴를 만들지, 발송 시 파일/주소록으로만 할지
2. **요금 충전/결제:** 전단AI 자체 결제 페이지 필요 vs 한줄로 관리자가 충전
3. **슈퍼관리자 관리:** 전단AI 고객사 생성/관리를 한줄로 슈퍼관리자에서 할지, 별도 관리자 만들지
4. **발신번호 등록:** 전단AI에서 직접 신청 가능하게 할지, 한줄로 관리 메뉴 안내만 할지
