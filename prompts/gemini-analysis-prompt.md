## 분석 요청사항

### 1. 기능 목록 추출
모든 기획서를 분석하여 개발해야 할 기능을 계층 구조로 정리해주세요.

```
예시:
1. 관리자 시스템
   1.1 사용자 관리
       - 사용자 등록
       - 사용자 조회
       - PW 초기화
       ...
   1.2 전송 관리
       ...
2. 사용자 시스템
   ...
```

### 2. 데이터 엔티티 도출
기획서에서 언급된 모든 데이터 테이블/엔티티를 추출해주세요.
- 테이블명
- 주요 컬럼
- 다른 테이블과의 관계

### 3. 비즈니스 로직 정리
복잡한 비즈니스 로직을 정리해주세요:
- 발송 프로세스 (즉시/예약/분할)
- 카카오톡 실패 시 대체 문자 발송 로직
- 템플릿 검수 프로세스 (등록→검수요청→승인/반려)
- 수신거부 처리 로직
- 머지(변수 치환) 로직

### 4. 누락/모순점 발견
기획서 간에 불일치하거나 누락된 부분이 있는지 찾아주세요:
- 화면 A에서는 있는데 화면 B에서는 빠진 기능
- 관리자와 사용자 간 데이터 불일치
- 정의되지 않은 상태값이나 코드값
- 구현 시 추가로 필요한 정보

### 5. API 엔드포인트 도출
기획서 기반으로 필요한 REST API 목록을 추출해주세요:
- HTTP Method
- Endpoint
- 설명
- 필요 파라미터

### 6. 2019년 → 2026년 업데이트 필요사항
기획서가 2019년에 작성되었습니다. 현재 시점에서 변경/추가가 필요한 부분:
- 카카오 비즈메시지 API 변경사항
- 법규/규제 변경 (발신번호 사전등록제, 개인정보보호법 등)
- 신규 채널 추가 고려 (RCS 등)
