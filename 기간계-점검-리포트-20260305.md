## 🔴 CRITICAL (P0) — 상용화 차단 결함 11건

### 【보안】

#### P0-S1. Git에 커밋된 .env 파일 (프로덕션 시크릿 노출)
- **위치:** `packages/backend/.env`
- **문제:** JWT_SECRET, ANTHROPIC_API_KEY, MYSQL_PASSWORD, OPT_OUT_080_TOKEN 등 실제 프로덕션 시크릿이 Git 저장소에 노출
- **CLAUDE.md 위반:** #1 하드코딩 금지
- **조치:**
  1. `git filter-branch`로 Git 히스토리에서 완전 제거
  2. 노출된 모든 시크릿 즉시 로테이션
  3. `.gitignore`에 `.env` 추가 확인

### 【인증】

#### P0-A1. JWT_SECRET 기본값 하드코딩
- **위치:** `auth.ts:23,40` / `database.ts:38`
- **문제:** `'dev-only-secret'` 기본값 → 환경변수 미설정 시 위조 토큰 생성 가능
- **조치:** 미설정 시 즉시 에러 throw (fail-fast)

#### P0-A2. 임시 비밀번호 암호학적 약점
- **위치:** `admin.ts:146-170`
- **문제:** `Math.random()` 사용(예측 가능), 8자(짧음), SMS 평문 발송(통신사 로그 노출)
- **조치:** `crypto.randomBytes()` + 웹 기반 재설정 링크

#### P0-A3. 슈퍼관리자 세션 무한 유효
- **위치:** `auth.ts:43-46`
- **문제:** 세션 체크 완전 생략 → 토큰 탈취 시 24시간 무제한 접근
- **조치:** Redis 토큰 블랙리스트 + 슈퍼관리자도 세션 검증

### 【발송 — 직접발송】

#### P0-C1. PostgreSQL↔MySQL 크로스DB 트랜잭션 부재
- **위치:** `campaigns.ts:1872-2107`
- **문제:** PostgreSQL INSERT 성공 → MySQL INSERT 실패 시 캠페인 레코드만 남고 메시지 미발송. 선불 차감/환불 불일치 가능
- **조치:** BEGIN/COMMIT 래핑 + MySQL 실패 시 PostgreSQL ROLLBACK

#### P0-C2. 전화번호 정규화 불일치
- **위치:** `campaigns.ts:1787-1957`
- **문제:** 커스텀 메시지 맵 저장 시 `-` 제거, 조회 시도 `-` 제거 → 이중 제거/불일치 가능
- **조치:** `normalizePhone()` 함수 통일 (`/\D/g` 사용)

#### P0-C3. 분할발송 시간 오버플로우
- **위치:** `campaigns.ts:1979-1995`
- **문제:** 1000명 분할 10명/분 → 100분 후 다음날로 넘어감 → 일일 발송 제한 위반 가능
- **조치:** send_start_hour/send_end_hour 범위 내 관리

### 【발송 — AI 맞춤】

#### P0-C4. 예약 발송 SQL 동적 생성 오류
- **위치:** `campaigns.ts:1072`
- **문제:** `sendTime` 값을 SQL 문자열로 직접 삽입 → 파라미터 매칭 불일치 → 예약 발송 MySQL INSERT 실패
- **조치:** 항상 파라미터화된 쿼리 사용

### 【발송 — 테스트】

#### P0-C5. 테스트 발송 추적 불가
- **위치:** `campaigns.ts:663-700`
- **문제:** `app_etc1='test'` + `bill_id=userId`로 추적하나, userId 미설정 시 조회 불가. 실패 건 별도 기록 없음
- **조치:** 테스트 발송 전용 request_uid(UUID) 발급

### 【발송 — 스팸필터】

#### P0-C6. 메시지 해시 매칭 실패 (빈 콘텐츠)
- **위치:** `spam-filter.ts:408-449`
- **문제:** 메시지 내용이 공백/특수문자만 있으면 해시 "" 반환 → 복수 active 테스트 중 매칭 불가 → TIMEOUT 오인
- **조치:** 해시 실패 시 대체 매칭 로직 강화

### 【QTmsg】

#### P0-Q1. MySQL 테이블명 SQL Injection
- **위치:** `campaigns.ts:676, 1070, 2037` / `admin.ts:169`
- **문제:** 템플릿 리터럴로 테이블명 삽입 → 악의적 설정 시 SQL Injection 가능
- **조치:** ALLOWED_TABLE_REGEX 화이트리스트 검증
